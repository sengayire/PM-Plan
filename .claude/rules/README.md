# _pm-plan Rules — Story & Task Standards

This directory contains templates and standards for product requirements documentation.

## Rules

- **[stories.md](stories.md)** — User story template with acceptance criteria and dependencies
- **[tasks.md](tasks.md)** — Implementation task template with test patterns and DOD

## Key Constraints

- Every story must have structured acceptance criteria (Backend, Frontend, Database, QC sections)
- Every story must have at least 1 implementation task
- Tasks must include code examples, test cases, and an acceptance checklist
- All stories link to Jira tickets (FL-N format)
- Dependency chains documented with arrow notation (blocks, blocked-by, related)
